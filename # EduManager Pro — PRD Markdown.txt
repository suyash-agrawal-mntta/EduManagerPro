## 1. Executive Summary & Vision

> **EduManager Pro** is a one-time-purchase, all-in-one preschool management and marketing platform built natively for Indian preschool owners. It is the **world's first school management system** to combine operational management with a complete social media marketing engine — all in one product at a single lifetime price with no recurring charges, no subscriptions, and no per-user fees ever.

### 1.1 Problem Statement

Indian preschool owners face three simultaneous crises that no single tool addresses today:

- **Administrative overload** — attendance, fees, parent communication, staff management, and compliance are managed across notebooks, WhatsApp groups, and disconnected Excel files with zero integration
- **Marketing paralysis** — 80% of preschool owners know they need to post on Instagram and Facebook but do not know what to write, how to design posts, or when to publish
- **Expensive and irrelevant software** — existing tools cost ₹10,000–₹50,000/year, are built for the US/UK market, address only one function, and require recurring subscription payments that school owners deeply resent

### 1.2 Unique Differentiators

| Feature | Description |
|---------|-------------|
| Social media engine | 34 auto-personalised post templates + AI image prompts + Canva guide — no competitor has this anywhere in the world |
| WhatsApp-first architecture | All notifications, alerts, communications, receipts, and updates via WhatsApp — matching Indian parent behaviour |
| School-type adaptive | Montessori and Kindergarten modes — all templates, trackers, and reports adapt automatically |
| Daily photo sharing | One-tap classroom photo to WhatsApp for all parents — the #1 daily task Indian parents expect |
| Birthday management | Auto-alert + personalised WhatsApp + social media post on every child's birthday — zero misses |
| Auto-generated Parent Handbook | Professional PDF handbook auto-generated from school profile in one click |
| ECA Revenue Engine | Complete post-school-hour activity management — additional ₹1.5–₹2.5L/month revenue for a 50-student school |
| India-native payments | UPI QR receipts, NEFT, cash/cheque log, Tally CSV export |
| One-time price forever | ₹14,990 once — replaces ₹20,000–₹75,000/year in annual subscriptions |

## 2. School Type Configuration

EduManager Pro supports **TWO school types only**. Schools that blend both methods (Integrated) select Montessori and use subject-grade fields in addition to Montessori material tracking.

### 2.1 School Types

| Type | Description |
|------|-------------|
| **Montessori** | Child-led work cycles · Material progression tracking (5 areas: Practical Life, Sensorial, Language, Mathematics, Cultural) · Observation-based assessment · MNTTA ECCE curriculum alignment · NEP 2020 outcome mapping · Child Portfolio PDF report · Montessori-specific social media templates |
| **Kindergarten (Traditional)** | Structured teacher-led sessions · Subject timetable (English, Maths, EVS, Hindi, Art, Music, PE) · Unit plan templates · Grades-based assessment · Traditional report card · Activity-focused social media templates · CBSE/ICSE-aligned terminology |

### 2.2 School Type Rules

- Selected **once** during 5-minute onboarding
- Displayed as a badge throughout the entire app
- **Can be changed** in Settings at any time — all existing data is preserved
- Changes which templates, reports, curriculum screens, and social media language are shown throughout the app

## 3. Pricing Architecture & Licence Management

### 3.1 Single-School Pricing Tiers

| Tier | Price | Inclusions |
|------|-------|-----------|
| **Early Bird** | ₹7,990 one-time | First 100 buyers only · All 19 modules · Lifetime access · Email support 48hr · Live counter on sales page |
| **Standard** | ₹14,990 one-time | Permanent price after early bird · All 19 modules · Lifetime access · WhatsApp support 24hr · 1 free 30-min onboarding call |
| **Pro + Setup** | ₹24,990 one-time | All Standard + GoHighLevel CRM configured + n8n automation workflows set up + MNTTA expert 3-hour setup session + 3 months priority support |

> **CRITICAL ARCHITECTURE DECISION:** All three tiers get all 19 modules. Differences are in support and setup services only — NOT in software features. Build one webapp. Read `plan_tier` from `_licence` tab at startup to show appropriate support contact and upgrade options.

```javascript
// Licence check — run at app startup
function getLicenceTier() {
  const sheet = SpreadsheetApp.getActive().getSheetByName('_licence');
  return sheet.getRange('B2').getValue(); // returns: early_bird | standard | pro
}
```

### 3.2 Licence Record Fields

The `_licence` tab stores:
```
school_id | owner_name | email | phone | plan_tier | purchase_date | payment_mode | 
payment_reference | upgrade_history_json | support_entitlement | onboarding_call_status | 
school_type | active
```

### 3.3 Plan Upgrade Flow

1. Owner → Settings → My Plan → View current plan and benefits
2. Click "Upgrade" → See delta benefit description
3. Pay difference (auto-calculated: e.g. ₹14,990 – ₹7,990 = ₹7,000 upgrade fee)
4. Enter UPI/NEFT payment reference
5. `plan_tier` updates instantly in `_licence` tab
6. WhatsApp confirmation sent to owner

**Rules:**
- No downgrades permitted — communicated clearly at purchase
- 7-day refund window if fewer than 20% of features used (tracked via `_audit_log`)
- Licence transfer allowed once per licence with documentation — ₹999 transfer fee

### MODULE 3 — Admissions CRM & Lead Management

```
Parent Name   → Text input
Phone         → Tel input (auto-format Indian number)
Child Age     → Number input (months or years)
Source        → Dropdown: WhatsApp | Instagram | Facebook | Walk-in | Referral | Other
```

Save in 10 seconds. Lead created in CRM. Full details filled later from the lead record.

### MODULE 4 — Attendance Tracking

#### DEFAULT BEHAVIOUR — Mark All Present First (MUST HAVE)
**Business rule:** Attendance screen opens with ALL students already marked PRESENT by default. Teacher only taps names to change to Absent, Late, or Half-Day. Do NOT build a system that requires marking each child individually. In a 20-child classroom, typically only 1–3 changes are needed daily.

#### Attendance Record
```
att_id              → ATT-NNN
student_id          → FK: students
date                → Date
status              → Present | Absent | Late | Half-Day (default: Present)
check_in_time       → Time (set when QR code scanned or manual check-in)
health_status       → Healthy | Fever | Rash | Cough | Sent-Home (set during gate check)
health_temp         → Decimal (°C or °F, configured in settings)
marked_by           → FK: staff (who marked the attendance)
```

#### QR Code Check-in (Optional)
- Generate unique QR per student (displayed in student profile)
- Teacher scans on arrival → timestamp recorded → auto-marks Present
- Overrides the default-present state with actual check-in time

#### Morning Health Screening Log (MUST HAVE)
After marking attendance, teacher records gate health check:
```
screen_id           → HSC-NNN
student_id          → FK
date                → Date
temperature         → Decimal (°C or °F)
status              → Healthy | Fever | Rash | Cough/Cold | Sent-Home
sent_home           → Boolean
notes               → String
recorded_by         → FK: staff
```
**Business rule:** If `status = Sent-Home`, auto-send WhatsApp to parent immediately:  
`"We noticed [child name] had [health issue] today and asked you to take them home. Please rest for 48 hours and return when fully well. — [School Name] ([Phone])"`

#### WhatsApp Arrival Alert
- Trigger: student check-in event
- n8n sends WhatsApp to parent: `"[Child name] has arrived safely at [School Name] at [time]. Have a great day! 🌱"`
- Toggle on/off per parent preference (stored in `families` tab)

#### Absence Notification
- Trigger: AppScript time-based trigger at 9:30 AM
- If student NOT checked in by 9:30 AM → WhatsApp to parent: `"We have not seen [child name] today. Is everything okay? Please let us know. — [School Name]"`
- Can be disabled per student (if parent pre-notified)

#### Pickup Log — Safety Critical (MUST HAVE)
```
pickup_id           → PKP-NNN
student_id          → FK
att_id              → FK (links to today's attendance record)
collector_name      → String (from authorised pickup list)
relationship        → String
time_of_collection  → Timestamp
verified_against_authorised_list → Boolean
staff_id            → FK (staff member who released child)
notes               → String
```
**Business rule:**  
1. Before logging pickup, system displays the authorised pickup list for that student (names + photos from M2 family profile)
2. Staff confirms identity visually and taps the correct person from the list
3. If person is NOT on authorised list → system shows warning and requires owner override
4. Child must never be released without a pickup log entry

#### Late Pickup Tracking
- If child not picked up by (school close time + late pickup threshold from M1 config):
  - System auto-calculates late pickup fee (per 15 minutes × fee rate from M1 config)
  - Fee added to next month's fee collection screen
  - WhatsApp to parent: `"[Child name] was picked up at [time], [X] minutes after school close. A late pickup charge of ₹[amount] has been added to your account."`

#### Teacher Login — Limited Access
**Business rule:**
- Teacher logs in with their own limited-role credentials (email set in their staff profile)
- Sees ONLY their assigned classroom students
- Can ONLY mark attendance and add curriculum observations (M10)
- Cannot see: fee records, CRM leads, staff salaries, management reports, other classrooms

#### Substitute Teacher Assignment (MUST HAVE)
**Trigger:** Teacher marks themselves absent OR owner marks a teacher absent  
**Flow:**
1. System shows: `"⚠️ [Classroom Name] is uncovered. [Teacher Name] is absent today."`
2. Owner sees list of available staff (not currently assigned elsewhere)
3. Owner taps to assign substitute to the classroom for the day
4. Ratio dashboard updates with substitute in place
5. Auto-generate WhatsApp to parents of affected classroom:  
   `"[Teacher Name] is unwell today. [Substitute Name] will be with [Classroom Name]. — [School Name]"`

#### Staff Attendance
- Separate daily check-in/check-out for staff
- Leave request and approval workflow
- Leave balance tracker: CL (Casual Leave) · PL (Privilege Leave) · SL (Sick Leave) per Indian labour norms
- Monthly staff attendance summary

#### Monthly Attendance Reports
- Per student: days present / absent / late / health-sent-home / attendance %
- Annual attendance register PDF (format aligned to school inspection requirements)
- Bulk PDF generation for all students

### MODULE 5 — Fee Management

#### Defaulter Dashboard
- Outstanding fees by student with filter by batch/classroom
- Total outstanding amount (school-wide)
- Months overdue per student
- One-tap WhatsApp to individual parent
- Bulk WhatsApp to all defaulters (batch-selectable)

#### Tally CSV Export
- Monthly fee collection data in Tally-compatible CSV format
- Eliminates manual double-entry for school accountant
- Export button in Reports section

#### Annual Fee Report
- Fee collected vs expected per month (bar chart)
- Year-to-date collection total
- Outstanding total
- Revenue by fee head breakdown
- Export PDF for CA/accountant

- Holiday notice
- Exam/assessment schedule

- Fee reminder

### 5.2 AppScript DAL — 10 Mandatory Coding Rules

1. **Every read and write routes through AppScript DAL functions** — `getStudents()`, `addFeeRecord()`, `markAttendance()` etc. Never call `sheet.getRange().setValue()` directly from UI handlers.

2. **Every write appends to `_audit_log`:** `timestamp | user_role | action (INSERT/UPDATE/DELETE) | table_name | row_id | field_changed | old_value | new_value`

3. **Every row has a unique auto-generated ID in Column A** — format: `PREFIX-YYYYMMDD-NNN` (e.g. `STU-20260615-001`). Never use row numbers as identifiers. Row numbers change when rows are sorted.

4. **Soft deletes only** — `deleteRecord()` sets `deleted = TRUE` and `deleted_at = timestamp`, then moves the complete row as a JSON snapshot to the `_deleted` tab. Never call `sheet.deleteRow()`.

5. **No formulas in data tabs** — raw values only. All calculations (age from DOB, fee totals, attendance %) in AppScript code. Sheets store computed results as plain values.

6. **No merged cells in any data tab** — merged cells break `sheet.getValues()` and all CSV exports. Merged cells only allowed in `_config` for visual display, read via Named Ranges only.

7. **Continuous data from Row 2** — Row 1 is always the header row. Data is continuous with no empty rows. AppScript uses `sheet.getLastRow()` to find append position. Empty rows break `getLastRow()`.

8. **Foreign keys enforced in AppScript before every write** — before inserting a fee record, verify `student_id` exists in `students` tab. Before deleting a student, check for linked records and prompt owner for confirmation.

9. **All tabs hidden at deployment** — `sheet.hideSheet()` on every data tab. Owner sees only a blank Google Sheets file if they navigate to the URL. The web app is the only interface.

10. **`LockService.getScriptLock()` with 10-second timeout before every write** — prevents concurrent access corruption when two users submit forms simultaneously.

## 9. Data Protection, Integrity & Backup

### 9.1 Five-Layer Protection Model

| Layer | Implementation |
|-------|---------------|
| **Layer 1: Sheet protection** | Every tab protected via `sheet.protect().setPassword('generated-secret')`. Users editing directly see: "This range is protected." AppScript bypasses via service account internally. End users cannot bypass. |
| **Layer 2: DAL wall** | All reads and writes through typed AppScript DAL functions. No module calls `sheet.getRange().setValue()` directly. The GUI never touches the Sheet. |
| **Layer 3: Input validation** | Before every write: required fields present · text max length · valid date format (DD/MM/YYYY) · numeric in valid range · FK references exist. Invalid data rejected with clear GUI error message. |
| **Layer 4: Soft deletes only** | `deleteRecord()` sets `deleted=TRUE + deleted_at` and moves JSON snapshot to `_deleted` tab. Owner restores from Settings → Deleted Records in 30 seconds. Never calls `sheet.deleteRow()`. |
| **Layer 5: Audit log** | Every write appends to `_audit_log`: timestamp · user_role · action · table · row_id · field_changed · old_value · new_value. Append-only — existing rows cannot be edited even by AppScript. |
### 9.2 Three-Copy Backup System (3-2-1 Rule)

| Copy | Schedule | Storage | Retention | Recovery Time |
|------|----------|---------|-----------|---------------|
| **Copy 1 — Daily** | 2:00 AM IST every night | Same Google Drive account — "EduManager Backups" folder | Last 30 copies | 2 minutes |
| **Copy 2 — Weekly** | Sunday 3:00 AM IST | Separate MNTTA backup Google Drive account (independent of school account) | Last 12 weekly exports per school | 15–30 minutes |
| **Copy 3 — Monthly** | 1st of month, 8:00 AM IST | Owner's phone via WhatsApp (monthly summary PDF) | Permanent (owner's device) | Summary data only |

**Copy 1 implementation:**
```javascript
function dailyBackup() {
  const ss = SpreadsheetApp.getActiveSpreadsheet();
  const date = Utilities.formatDate(new Date(), 'Asia/Kolkata', 'yyyy-MM-dd');
  const backupFolder = DriveApp.getFoldersByName('EduManager Backups').next();
  DriveApp.getFileById(ss.getId()).makeCopy(`school_backup_${date}`, backupFolder);
  // Delete backups older than 30 days
  cleanOldBackups(backupFolder, 30);
  // Log backup
  logBackup('daily', 'success');
  // Send WhatsApp confirmation to owner via n8n
  notifyOwnerBackupComplete(date);
}
```

**Backup health check:** After each backup: verify file created · file size reasonable · row counts consistent with previous backup. If any check fails → WhatsApp alert to owner AND MNTTA support.

**Owner WhatsApp confirmation (auto-sent 2:05 AM IST):**  
`"EduManager Pro backup complete for [School Name] — [date] 2:00 AM. [N] student records, [N] fee records. Backup ID: [ID]. All data safe. ✅ — EduManager Pro"`

### 9.3 Recovery Scenarios

| Incident | Severity | AppScript Defence | Recovery Method |
|----------|----------|------------------|----------------|
| Accidental cell edit | Low | Sheet protection blocks edit entirely | No action needed — protection worked |
| Row sort scrambles data | Medium | Row IDs (not row numbers) keep all relationships intact | File → Version History → Restore. Takes 60 seconds. |
| Student/staff record deleted | High | Soft delete — JSON snapshot in `_deleted` tab | Settings → Deleted Records → Restore. 30 seconds, zero data loss. |
| Entire Sheet tab deleted | Critical | Tab hidden from tab bar — owner doesn't know it exists | Edit → Undo (if immediate) or Version History restore or Copy 1 backup |
| Google account hacked/locked | Critical | Copy 2 (weekly CSV) in MNTTA backup account is untouched | MNTTA imports CSV to new account. Max data loss: 1 week |
| Formula/named range corruption | Medium | AppScript nightly health check validates all named ranges | AppScript auto-repairs from `_config` master list. WhatsApp alert sent. |

Differences are in support and setup services, NOT in software features.

The app reads `plan_tier` from `_licence` tab at startup and shows:
- Appropriate support contact (email for Early Bird, WhatsApp for Standard and Pro)
- Onboarding call booking link (Standard and Pro only)
- GHL/n8n setup status (Pro only)

### 10.3 Licence Delivery Flow (When School Purchases)

1. MNTTA receives payment confirmation
2. MNTTA creates Google Sheet workbook from master template
3. Set `plan_tier` in `_licence` tab: `early_bird` or `standard` or `pro`
4. Set `school_type` in `_config`: `Montessori` or `KG`
5. Deploy AppScript Web App and note the URL
6. Send WhatsApp to school owner with: unique web app URL + onboarding video link + WhatsApp support number
7. For Pro tier: schedule 3-hour setup session within 48 hours of purchase
