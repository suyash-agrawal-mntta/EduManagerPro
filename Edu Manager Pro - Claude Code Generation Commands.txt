Three practical tips for each Claude Code run
- First, always upload the PRD FINAL document at the start of every run — Claude Code cannot remember previous sessions, so the PRD is its complete reference each time.
- Second, at the end of each run, ask Claude Code: "Export the complete AppScript project as a ZIP and write a handoff note summarising what was built and what the next run should continue from." That handoff note becomes the brief for the next session.
- Third, for Run 2 onwards, also upload the ZIP from the previous run so Claude Code can see the existing code structure before adding to it.
